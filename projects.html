2015-09-05

The purpose of this website was to allow my friend Dave to post content, and link to the content posted on his facebook wall to allow him to reach a wider audience. The easiest way to post content is to edit the html directly, but that would require Dave to have access to the website's administrative account. Not that I don't trust him, but handing out admin credentials is not a viable long-term solution.

The next step is to add a feature to the website that will allow visitors to sign up for accounts and log in on subsequent visits. The main goal will be to allow a certain role to publish content, but users assigned to other roles could have additional privileges eventually. I'll also need a way to manage user roles, since I probably don't want users deciding their own roles when they sign up.

I've started implementing a MySQL database. I have an initial design for users, roles, permissions, etc implemented, so the next step is adding the server and client-side functionality that allows people to sign up and log in. Should be interesting.
